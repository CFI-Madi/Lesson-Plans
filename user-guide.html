CFI Training App Guide

This guide covers the day-to-day workflows for managing students, lessons, endorsements, and backups in the aviation training app.

Getting Started

  1. Use Add Student in the top bar to create a student profile.
  2. Select the student from the student dropdown.
  3. Open Lessons to begin marking progress.
  4. Use Flight Log after each lesson to record flight time and notes.
The dashboard becomes more useful as soon as you log flights, mark lesson signoffs, and record milestones.

Managing Students

  • Add Student creates a new training record and makes that student active.
  • Manage Student lets you switch, edit profile details, or delete a student.
  • Profile fields like start date, certificate number, and medical expiration improve dashboard insight and endorsement generation.

Lessons And Progress

  • Open a lesson from the sidebar or search bar.
  • Use the lesson competency pills to move from introduced to signed off.
  • Signed-off lessons automatically contribute to dashboard readiness and lesson completion metrics.
  • Use lesson notes and checklists for student-specific coaching details.

Flight Log

  • Use Log Flight Session to add a new entry.
  • Record aircraft, lesson, landings, ground time, flight time, and notes.
  • You can edit or delete entries from the Flight Log table.
  • Dashboard totals update automatically after each change.

Endorsements

  • Use the tracker to mark required endorsements complete.
  • Use the generator to build endorsement text from FAA guidance.
  • Copy the generated endorsement text into your preferred document or logbook workflow.

Reading The Dashboard

  • Stats row: flights, time, lesson progress, and signoffs.
  • Next lesson: suggested lesson based on current progress.
  • Readiness cards: training progress toward checkride preparation.
  • Recent activity: latest flights and recently visited lessons.
  • Personal minimums: student-specific operational limits.

Backup And Restore

  1. Open Settings.
  2. Choose Export Backup to save a recovery file.
  3. Store the exported .json file somewhere safe.
  4. Use Import / Restore to merge a saved backup into the app.

The app now keeps a local recovery snapshot before restore so accidental imports are easier to recover from.

Best practice: export a backup before major updates, before restoring another file, and at least once every couple of weeks during active training.

Troubleshooting

  • If the app warns that some saved data was repaired, review your students and restore the latest backup if needed.
  • If the Dashboard or Flight Log looks unavailable, refresh the page first. The app keeps a vanilla fallback for those views.
  • If you use a hosted copy, open it over http:// or https:// rather than directly as a local file path.
